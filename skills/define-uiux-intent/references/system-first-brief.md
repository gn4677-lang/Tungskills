# System-First Brief

Use this for AI products, editors, dashboards, developer tools, creation tools, or feature-heavy surfaces.

## Core System Questions

- What is the central object or task?
- Is the interface file-centered, task-centered, agent-centered, content-centered, workflow-centered, or people-centered?
- What primitives should users learn once and reuse?
- Which features are overlapping concepts that can become modes, states, or variants?
- Which complexity is necessary for power users, and which complexity is accidental?
- What defaults shape user behavior?
- What should remain stable as the product grows?

## Feature Unification Pass

For each feature or section, ask:

- Is this a separate concept or a mode of an existing concept?
- Does it add power without adding a new mental model?
- Can its entry point be merged with a more important path?
- Does it clarify the system or make users choose between similar things?

## Task-Centered Check

Rewrite object-centered labels into task-centered intent:

| Object-centered | Task-centered |
| --- | --- |
| Files | What the user is trying to change |
| Charts | What decision the user needs to make |
| Agents | What work is being delegated |
| Settings | What behavior the user wants to control |
| History | What changed and what can be recovered |

Prefer task-centered organization unless the user's domain requires object navigation.

## Common System Models

Use these as starting points, not templates:

- AI coding dashboard: workflow-centered around runs, phases, evidence, interventions, and risk.
- AI analyst or research tool: task-centered around question, sources, reasoning/evidence, answer, and review/export.
- Editor or creation tool: object-plus-task centered around canvas/document, tools, selection, history, and output.
- SaaS operations dashboard: role/task-centered around monitoring, exceptions, decisions, and repeated actions.
- Landing page: communication-centered around audience, literal offer, proof, workflow, trust, and conversion.
- Marketplace or catalog: object-centered around discovery, comparison, trust, and transaction.
