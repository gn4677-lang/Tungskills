# Anti-Slop Direction Examples

Use this when "don't make it generic" is too vague to guide the design.

## 1. Generic SaaS landing page vs literal product offer

Bad direction:

- giant headline about "unlocking productivity"
- three feature cards
- testimonial strip
- CTA footer
- no concrete view of the actual product

Why it is slop:

- the layout is doing the talking because the product story is missing

Better direction:

- literal subheadline stating what the product does
- one believable screenshot or workflow frame above the fold
- task-based proof section instead of generic feature cards

## 2. Fake dashboard vs real operational surface

Bad direction:

- charts and stats tiles used as decoration
- random metrics with no obvious operator action
- pretty panel layout but no real decision path

Why it is slop:

- it mimics "data product" aesthetics without supporting an actual workflow

Better direction:

- queue, exception list, recent actions, or status table tied to real operator decisions
- metrics only when they help prioritize or verify work

## 3. Card wall vs task-centered app

Bad direction:

- everything becomes a card
- equal visual weight for all information
- no obvious starting point or primary action

Why it is slop:

- card repetition substitutes for information structure

Better direction:

- use layout hierarchy to show what users read first, decide next, and act on last
- reserve cards for genuinely repeated objects, not whole page composition

## 4. AI-chat shell vs domain-fit workflow

Bad direction:

- default chat sidebar + message area regardless of product
- assistant framing even when users really need a workspace, editor, or decision console

Why it is slop:

- the shell comes from the model, not from the task

Better direction:

- choose a system model that fits the job: workflow board, editor, inspector, queue, canvas, catalog, or command surface
- use chat only when conversational exchange is actually the core interaction

## 5. Decorative premium vs trustworthy product

Bad direction:

- glow, blur, glass, oversized shadows, dramatic gradients
- premium feeling with weak proof and weak copy

Why it is slop:

- visual mood is replacing trust and product comprehension

Better direction:

- restrained visual system
- trust proof at the moment of doubt
- product-specific content, labels, and examples

## 6. Minimal emptiness vs intentional reduction

Bad direction:

- very little copy
- sparse empty space
- no explanation of what users can do next

Why it is slop:

- "minimal" becomes lack of decision-making

Better direction:

- reduce only what is unnecessary
- keep the core task, trust cue, and next action explicit

## Quick rule

If the design could swap brand name and still fit ten unrelated startups, it probably needs stronger product signal, stronger task logic, or stronger proof.
