# Heuristic Review Rubric

Use this as a structured expert-review pass. Findings are risks, not proof from real users.

## Heuristics

- System status: Users can tell what is happening now.
- Real-world match: Language and concepts fit the user's domain.
- User control: Users can cancel, undo, edit, retry, or leave safely.
- Consistency: Similar things look and behave similarly.
- Error prevention: Risky actions are constrained before errors occur.
- Recognition over recall: Options and context are visible when needed.
- Efficiency: Novices can proceed and experts can move quickly.
- Minimalism: Every visible element earns its space for the task.
- Error recovery: Messages explain the issue and next step.
- Help: Complex or unfamiliar tasks have accessible guidance.

## Finding Format

- Severity: must fix / should fix / human judgment needed.
- Evidence: what was observed.
- User impact: why it matters.
- Suggested repair: what to change.

## Limits

Do not treat heuristic review as usability testing. If the risk depends on user comprehension, recommend real user feedback.
