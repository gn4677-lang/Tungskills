# Visual Example Index

Use this when text-only anti-slop rules are too abstract and a paired visual comparison would help.

If image viewing is available, inspect the SVG asset directly. If not, read the SVG file as text; each asset includes a `<title>` and `<desc>` explaining what to notice.

## How to use the gallery

1. Name the suspected failure family first.
2. Inspect the matching asset.
3. Compare the live UI or screenshot against the left `bad` and right `better` panels.
4. Use the example to sharpen the finding, not to force a style clone.

## Important boundary

These visuals teach contrast, not aesthetic law. Do not say "this uses cards/gradients/chat, therefore it is slop." Ask whether those patterns are replacing product meaning, workflow clarity, trust, or truthful state communication.
